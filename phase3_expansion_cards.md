### 牌 #013：合夥人攤牌

#### ① 情境描述

**牌型：** 事件型（變體 A）
**格式：** 級聯型（選 A/B 後，下季首位牌替換為子牌 #013-A1 或 #013-B1）
**觸發時機：** Y2–Y4 任意決策季，現金動脈 > 50%

> 你的合夥人約你週末談話。他臉色凝重：
> 「我覺得公司方向不對。你太保守了，這樣下去會被市場淘汰。
> 我認為我們應該全力衝線上業務，砸三百萬進去。你如果不同意，
> 我想把我的股份賣掉，去做自己的事。」
> 他負責公司 40% 的核心技術架構。

#### ② 當下數字

```
可動用現金      $400 萬
本季預計淨額    +$40 萬
現金動脈        71% 🟢
組織心跳        65 🟡
客戶根系        60 🟡
```

#### ③ 決策空間

**選項 A：同意合夥人方案，投 $300 萬衝線上業務**

```
├ 現金 -$300 萬
├ 組織心跳 +6（合夥人幹勁回來，團隊看到方向感）
├ 客戶根系 -4（線下客戶資源暫時被分散）
├ ⏳ 伏筆
├ 執行門檻：現金動脈 > 55%
└ 鎖定提示：若不符合 →「現金不足以支撐線上投資」
```

**選項 B：拒絕方案，接受合夥人離開，回購股份**

```
├ 現金 -$120 萬（股份回購）
├ 組織心跳 -10（核心人物離開，團隊震盪）
├ 客戶根系 -3（合夥人走了帶走部分技術信心）
├ ⏳ 伏筆
├ 執行門檻：現金動脈 > 40%
└ 鎖定提示：若不符合 →「現金不足以回購股份」
```

#### ④ 立即後果

**選 A：** 現金 $140 萬，動脈 25% 🔴，心跳 71，根系 56
**選 B：** 現金 $320 萬，動脈 57%，心跳 55，根系 57

**歸因標籤：**
```
選 A：現金動脈 ▼ 大「線上業務投資」/ 組織心跳 ▲ 中「合夥人回歸、方向確立」/ 客戶根系 ▼ 小「資源分散」
選 B：現金動脈 ▼ 中「股份回購」/ 組織心跳 ▼ 大「合夥人離開」/ 客戶根系 ▼ 小「技術信心受損」
```

#### ⑤ 伏筆種子

**選 A：** 延遲 +5 季
```
├ 樂觀結果（60% 機率）：線上業務起飛，季收入 +$80 萬，客戶根系 +12
└ 悲觀結果（40% 機率）：線上業務失敗，現金 -$100 萬（追加投入後止損），合夥人再次消沈，心跳 -8
系統隨機決定（首次引入隨機伏筆機制）
溢出處理：×0.7
```

**選 B：** 延遲 +4 季
```
├ 組織心跳 +8（團隊適應新架構，反而更團結——「留下的都是真的想做的人」）
└ 客戶根系 +5（沒有被線上業務分散，線下深耕見效）
溢出處理：×0.7
```

#### ⑥ 設計驗證（內部用）
```
覆蓋知識點：ROI（主要）、機會成本（主要）
設計特色：首張引入隨機伏筆結果的牌——模擬「投資不保證回報」
與其他牌互補：與 #008（跨界競爭者）互補——都涉及「要不要大幅轉型」
```

### 牌 #014：海外大單

#### ① 情境描述

**牌型：** 事件型（變體 A）
**格式：** 三路選擇
**觸發時機：** Y3–Y5 任意決策季，客戶根系 > 55

> 一封英文 email 躺在你信箱裡。東南亞一家大型企業集團想跟你合作，
> 第一筆訂單 $200 萬，但要求你在當地設立服務點。
> 業務主管興奮地說：「這是我們走出去的機會！」
> 財務主管皺著眉：「海外設點前期投入至少 $150 萬，回本要一年半。」

#### ② 當下數字

```
可動用現金      $380 萬
本季預計淨額    +$40 萬
現金動脈        68% 🟡
組織心跳        63 🟡
客戶根系        62 🟡
```

#### ③ 決策空間

**選項 A：接受訂單，設立海外服務點**

```
├ 現金 -$150 萬（設點成本）+ 應收帳款 +$200 萬（到帳需 2 季）
├ 組織心跳 -5（核心人員被派駐海外，本部人手吃緊）
├ 客戶根系 +3（有了國際客戶的招牌）
├ ⏳ 伏筆
├ 執行門檻：現金動脈 > 55%，組織心跳 > 50
└ 鎖定提示：「現金或團隊不足以支撐海外擴張」
```

**選項 B：婉拒海外訂單，專注國內深耕**

```
├ 現金 $0
├ 組織心跳 +3（團隊專注，不被分散）
├ 客戶根系 +5（把本來要投海外的精力全用在國內客戶維護）
├ ⏳ 伏筆
├ 執行門檻：無
└ 鎖定提示：N/A
```

**選項 C：接單但不設點——遠端服務 + 派駐短期團隊**

```
├ 現金 -$60 萬（短期派駐 + 遠端基礎設施）
├ 應收帳款 +$200 萬（到帳需 2 季，同選項 A）
├ 組織心跳 -2（比 A 的 -5 輕——不用長期外派）
├ 客戶根系 +1（有國際客戶但不如設點有說服力）
├ ⏳ 伏筆
├ 執行門檻：現金動脈 > 45%
└ 鎖定提示：若不符合 →「現金不足」
```

#### ④ 立即後果

**選 A：** 現金 $270 萬，應收帳款 $200 萬，動脈 48%，心跳 58，根系 65
**選 B：** 現金 $420 萬，動脈 75%，心跳 66，根系 67
**選 C：** 現金 $360 萬，應收帳款 $200 萬，動脈 64%，心跳 61，根系 63

#### ⑤ 伏筆種子

**選 A：** 延遲 +6 季
```
├ 應收 $200 萬中實際到帳 $160 萬（匯率+延遲扣損），+2 季入帳
├ 海外業務站穩：季收入 +$50 萬（持續），客戶根系 +10
└ 但每季固定支出 +$30 萬（海外營運成本，持續）
溢出處理：×0.7
```

**選 B：** 延遲 +4 季
```
├ 客戶根系 +8（國內客戶忠誠度提升，老客戶續約率 100%）
└ 季收入 +$20 萬（深耕帶來的穩定增長）
溢出處理：×0.7
```

**選 C：** 延遲 +4 季
```
├ 應收 $200 萬中實際到帳 $140 萬（遠端服務品質問題導致扣款更多）
├ 客戶不續約——無持續收入增加
└ 組織心跳 +3（團隊學到國際經驗，無長期外派負擔）
溢出處理：×0.7
```

#### ⑥ 設計驗證（內部用）
```
覆蓋知識點：ROI（主要）、現金流思維（次要）、競爭策略（次要）
設計特色：引入「持續性支出增加」的伏筆——海外業務不是一次性投資
選 C 教學點：半吊子的國際化比不做更浪費——投入不夠拿不到長期價值
與其他牌互補：與 #001（大客戶的好意）互補——都涉及「帳面好看 vs 實際可動用」
```

```
覆蓋知識點：機會成本（主要）、ROI（次要）

```

### 牌 #016：供應商壟斷

#### ① 情境描述

**牌型：** 危機型（變體 B）
**觸發時機：** Y2–Y4 任意決策季

> 你唯一的核心原料供應商通知你：下季起漲價 25%。
> 你打電話質問，對方很坦白：「市場就我們一家能做到你要的規格。
> 不滿意你可以找別人——但你知道沒有別人。」
> 你查了一下，他說的是事實。轉換供應商至少要 6 個月。

#### ② 當下數字

```
可動用現金      $360 萬
本季預計淨額    +$35 萬
現金動脈        64% 🟡
組織心跳        62 🟡
客戶根系        58 🟡
```

#### ③ 決策空間（危機型：主軸 + 附帶）

**主軸決策 A：接受漲價，同時啟動備用供應商開發**

```
├ 每季支出 +$35 萬（漲價，持續到備用供應商就位）
├ 現金 -$80 萬（備用供應商開發成本）
├ ⏳ 伏筆
├ 執行門檻：現金動脈 > 45%
└ 鎖定提示：若不符合 →「現金不足」
```

**主軸決策 B：強硬拒絕漲價，賭供應商讓步**

```
├ 本季無支出變化
├ ⏳ 伏筆（風險極高）
├ 執行門檻：無
```

**附帶決策（可選）：是否同步跟客戶協商轉嫁部分成本？**

```
├ 客戶根系 -4（客戶不開心但理解）
├ 效果：漲價對利潤的衝擊減半（支出只 +$18 萬/季）
├ 執行門檻：客戶根系 > 40
```

#### ⑤ 伏筆種子

**選 A：** 延遲 +5 季
```
├ 備用供應商就位，原供應商被迫降回原價
├ 每季支出恢復正常，且雙供應商格局長期降低風險
└ 客戶根系 +5（供應穩定帶來的交付可靠度）
```

**選 B：** 延遲 +2 季
```
├ 供應商不讓步，斷供一季
├ 客戶根系 -15（無法交貨，客戶暴怒）
├ 現金 -$100 萬（緊急從其他渠道高價採購）
└ 組織心跳 -5（團隊對管理層決策失去信心）
```

#### ⑥ 設計驗證（內部用）
```
覆蓋知識點：競爭策略（次要）、現金流思維（次要）、機會成本（主要）
設計特色：選 B 是「看起來強硬但其實魯莽」的陷阱，教「議價力要有替代方案支撐」
```

### 牌 #017：數據安全事件

#### ① 情境描述

**牌型：** 事件型（變體 A）
**格式：** 三路選擇
**觸發時機：** Y3–Y5 任意決策季

> 週一早上八點，IT 主管的電話把你從夢中叫醒。
> 「老闆，我們的客戶數據庫被入侵了。初步判斷有 3,000 筆客戶資料外洩。
> 媒體還不知道，但我們必須在 48 小時內決定怎麼處理。」
> 法務說：主動通報客戶是法律義務，但時機和方式由你決定。

#### ② 當下數字

```
可動用現金      $370 萬
本季預計淨額    +$35 萬
現金動脈        66% 🟡
組織心跳        60 🟡
客戶根系        65 🟡
```

#### ③ 決策空間

**選項 A：立即全面透明——主動公告 + 全額補償受影響客戶**

```
├ 現金 -$150 萬（補償金 + 資安升級 + 公關費用）
├ 客戶根系 -5（短期信心受損，但坦誠態度獲得尊重）
├ 組織心跳 +4（團隊看到公司有擔當）
├ ⏳ 伏筆
├ 執行門檻：現金動脈 > 50%
└ 鎖定提示：若不符合 →「現金不足以全額補償」
```

**選項 B：低調處理——僅通知受影響客戶，不公開聲明**

```
├ 現金 -$50 萬（最低限度資安修補 + 小額補償）
├ 客戶根系 -2（表面上影響小）
├ 組織心跳 -3（團隊對「掩蓋」做法有疑慮）
├ ⏳ 伏筆
├ 執行門檻：無
```

**選項 C：符合最低法律義務——通報主管機關 + 個別通知，不公開也不全額補償**

```
├ 現金 -$80 萬（資安修補 + 法律合規 + 個別通知成本）
├ 客戶根系 -3（受影響客戶知道了但處理態度不算積極）
├ 組織心跳 +1（至少沒有隱瞞——比 B 好一點）
├ ⏳ 伏筆
├ 執行門檻：無
```

#### ⑤ 伏筆種子

**選 A：** 延遲 +3 季
```
├ 客戶根系 +12（透明處理成為行業標桿，客戶信任反而比事件前更高）
└ 組織心跳 +5（「我在一家有良心的公司工作」）
```

## Phase 3 新增設計機制

### 隨機伏筆（牌 #013、#018 引入）

部分決策牌的伏筆結果不是固定的，而是系統在爆發時擲骰決定好壞。

**規則：**
- 隨機伏筆在牌的 ⑤ 層標注成功/失敗機率
- 系統在伏筆爆發當季隨機決定結果
- 玩家在預警階段（🔴）看到的仍然是模糊的——不知道是好是壞
- 覆盤時系統揭示「這個決策有 55% 的機率成功，你這次運氣……」

**設計意圖：** 模擬真實投資——不是所有冒險都有確定回報。有些決策做對了也可能失敗，做錯了也可能僥倖成功。這打破「選對選項就一定贏」的確定性思維。

**使用限制：** 18 張決策牌中最多 3 張有隨機伏筆。太多會讓玩家覺得「結果靠運氣不靠策略」。

### E-11 定時炸彈機制

E-11（資安漏洞預警）的「硬扛」選項不造成即時傷害，而是在未來某季隨機觸發一張系統事件。這是唯一一張「硬扛看起來沒事」的熵牌。

**設計意圖：** 教「不處理風險 ≠ 風險不存在」。CEO 經常忽略「沒出事就不管」的隱性風險。
